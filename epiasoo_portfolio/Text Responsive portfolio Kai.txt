# Road Accident Prediction

This project develops a machine learning framework to predict road accident severity (Fatal, Serious, Slight) using a Light Gradient Boosting Machine (LightGBM) model combined with a SMOTENC-based oversampling strategy to address class imbalance.

The study utilises real-world road traffic accident data from the UK Department for Transport (DfT) STATS19 dataset (2020–2024), integrating both collision-level and vehicle-level records to capture environmental, temporal, and driver-related factors influencing accident outcomes.

---

## Project Overview 

Road traffic accidents remain a significant public safety issue in the UK. Accurate prediction of accident severity can support:

- Evidence-based road safety policies
- Resource allocation for emergency services
- Proactive risk mitigation strategies
- Identification of high-risk conditions

This project builds a scalable and interpretable predictive pipeline that combines gradient boosting, advanced feature engineering, and explainable AI techniques.

---

## Methodology

The modelling pipeline includes:

### 1. Data Preprocessing
- Cleaning and integrating collision & vehicle datasets
- Handling missing values and outliers
- Feature encoding (ordinal & nominal categorical variables)
- Feature Engineering
- Temporal feature extraction (hour, day, seasonal effects)
- Cyclic encoding for time-based variables
- Feature aggregation at vehicle level

### 2. Handling Class Imbalance

Accident severity data is naturally imbalanced in whichh Fatal collisions are rare and Slight collisions dominate the dataset. To address this, SMOTENC (Synthetic Minority Over-sampling Technique for Nominal and Continuous features) was applied to the training data.

This approach:

- Generates synthetic minority samples
- Preserves categorical feature structure
- Prevents the model from biasing toward the majority class

Oversampling was applied only to the training set to avoid data leakage.

### 3. Model Training - LightGBM

- Multiclass Configuration
LightGBM was configured as a multiclass classifier to directly model the probability distribution across Fatal, Serious, Slight. Instead of decomposing into binary tasks, the model optimises a multiclass objective function, ensuring consistent probability estimation across all severity levels.

- Additional Setup

  - Class-balanced weighting applied
  - Explicit categorical feature specification
  - Parallel training enabled
  - Fixed random seed for reproducibility

Passing categorical features directly to LightGBM allowed it to apply specialised splitting strategies rather than relying on arbitrary numerical encodings.

### 4. Hyperparameter Optimisation

To ensure strong generalisation and balanced performance, hyperparameters were tuned using:

- Stratified 3-Fold Cross-Validation
- Optimisation Metric: Macro-F1
- Macro-F1 was selected because:
  - Accuracy is dominated by the majority class
  - Macro-F1 gives equal importance to all severity classes
  - It balances precision and recall for each class
- RandomizedSearchCV
  - 18 random hyperparameter configurations
  - 3-fold stratified validation
  - Total of 54 model fits
  - Parallel computation was used for efficiency
 
### 5. Hyperparameter Search Space

The search explored structural, learning, sampling, and regularisation parameters:
- Tree Structure
  - num_leaves: 31, 63, 95
  - max_depth: 7, 10
- Learning Dynamics
  - learning_rate: 0.03, 0.05, 0.10
  - n_estimators: 300, 500
- Sampling
  - subsample: 0.8, 1.0
  - colsample_bytree: 0.7, 0.9
- Leaf Constraints
  - min_child_samples: 40, 60
- Regularisation
  - reg_alpha: 0, 0.1
  - reg_lambda: 0, 0.1

This range allowed exploration of model expressiveness while maintaining computational feasibility.

### 5. Best Model Configuration

| Hyperparameter       | Selected Value |
|----------------------|----------------|
| num_leaves           | 95             |
| max_depth            | 10             |
| n_estimators         | 500            |
| learning_rate        | 0.05           |
| subsample            | 0.8            |
| colsample_bytree     | 0.7            |
| min_child_samples    | 40             |
| reg_alpha            | 0              |
| reg_lambda           | 0              |

Macro-F1 (CV mean): 0.4821 indicates balanced predictive ability across all three severity classes, despite the inherent difficulty and imbalance of the problem. Interestingly, explicit L1/L2 regularisation was not selected as structural constraints and subsampling already provided sufficient regularisation.

---

## Final Model Performance

The optimised LightGBM model achieved:

Accuracy: 0.6235

Strong Balanced Accuracy under skewed distributions

Improved Macro F1-score compared to non-resampled training

Better minority-class detection with SMOTENC

The comparison between models with and without oversampling highlights the importance of handling imbalance in real-world safety data.
